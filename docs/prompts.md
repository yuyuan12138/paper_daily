# Prompt设计

主要思路： 不要一次性分析完所有文章，最好的方式是分成不同的部分然后多次请求api分析

## 粗筛
你是一名严谨的科研助手。请根据我提供的论文标题、摘要和元信息，帮助我判断这篇论文是否值得深入阅读。

请严格输出以下内容：
1. 论文要解决的核心问题（1-2句）
2. 主要贡献（最多3点）
3. 方法的新意在哪里
4. 与我的研究方向的相关性（高/中/低），并解释原因
5. 潜在风险：可能只是工程堆叠、实验不充分、问题设定偏窄，或贡献不明显的地方
6. 阅读建议：值得精读 / 值得略读 / 可以跳过

要求：
- 不要空泛赞美
- 不要重复摘要原文
- 如果证据不足，请明确写“摘要中无法判断”
- 以审稿人视角给出谨慎判断

我的研究方向是：{research_interest}
论文信息如下：
{paper_metadata_and_abstract}

## 粗读
请把这篇论文拆解成一个适合快速阅读的结构化笔记。

输出格式：
1. 研究问题
2. 论文假设或动机
3. 方法总览
4. 模型/系统输入输出
5. 实验设置概览
6. 主要结果
7. 作者声称的优势
8. 论文的主要局限
9. 我如果只有10分钟，最值得看哪几节

要求：
- 用简洁学术语言
- 优先提炼逻辑主线
- 不要编造论文中没有的细节
- 如果某部分在原文不清楚，请直接指出

## 精读
请深入分析这篇论文的方法部分，目标是帮助我真正理解其技术设计，而不是只做摘要。

请按以下结构输出：
1. 方法整体流程（用步骤描述）
2. 每个核心模块的功能
3. 每个模块的输入、输出、关键操作
4. 作者为什么要这样设计，而不是采用更直接的方法
5. 与 baseline 相比，真正新增的部分是什么
6. 该方法最依赖哪些前提假设
7. 哪些地方可能是性能提升的主要来源，哪些地方可能只是辅助性改进
8. 如果让我复现，最关键的实现细节可能在哪些地方

要求：
- 优先解释“因果关系”和“设计动机”
- 避免只重复论文措辞
- 如果涉及公式，请解释公式的变量意义和它想表达的机制
- 如果文中信息不足，请明确指出无法确定

## 实验分析

请从批判性阅读角度分析这篇论文的实验部分。

请输出：
1. 作者想通过实验验证哪些核心 claim
2. 每组实验分别在支持什么论点
3. baseline 是否合理，是否强
4. 数据集和评测指标是否匹配研究问题
5. ablation 是否足够，缺了什么关键对照
6. 结果中最有说服力的证据是什么
7. 结果中最可疑或解释不充分的地方是什么
8. 如果你是审稿人，你会提出哪3个最关键的问题

要求：
- 不要只复述表格结果
- 重点分析“证据链是否闭合”
- 如果某个结论没有被充分支持，请明确指出

## 复现/落地
请从复现和工程实现角度整理这篇论文。

输出：
1. 复现该方法所需的最小要素清单
2. 数据需求
3. 模型结构需求
4. 训练流程需求
5. 推理流程需求
6. 关键超参数或训练技巧
7. 文中未明确说明、但复现很可能需要补充猜测的部分
8. 最可能导致复现失败的风险点
9. 若让我做一个简化版 baseline，应该保留哪些核心模块，删去哪些部分

要求：
- 区分“论文明确写了”和“基于经验推测”
- 不要把猜测说成事实

## 研究启发
请把这篇论文转化为研究启发，结合我的研究方向：{research_interest}。

请输出：
1. 这篇论文最值得借鉴的思想
2. 哪些设计可以迁移到我的任务中
3. 哪些设定不适合我的场景
4. 这篇论文有哪些明显局限，可以成为后续研究切入点
5. 可以延伸出的3个研究问题
6. 可以设计的2-3个改进方向
7. 如果把这篇论文作为 related work，我应该如何定位它

要求：
- 强调“迁移性”和“可研究性”
- 不要泛泛而谈
- 尽量提出具体、可执行的想法
